# pyproc 프로젝트 규칙

`pyproc`는 서버 없이 브라우저 탭에서 도는 **진짜 런타임 파이썬**을 제공하는 재사용 라이브러리다.
codaro / dartlab / xlpod가 공통으로 소비하는 웹 파이썬 런타임의 SSOT(single source of truth).

이 파일이 로컬 규칙의 SSOT다. `AGENTS.md`는 이 파일을 먼저 읽으라는 포인터로 유지한다.

## 정체성

브라우저에서 파이썬을 "노트북 한 셀"이 아니라 **운영체제처럼** 돌린다.

- **런타임(runtime)**: Pyodide 엔진을 얇게 감싸고, WASM 힙 접근 같은 교차 관심사를 능력 계약(capability) 뒤에 캡슐화한다.
- **복원 리액티브(reactive)**: 실행 경계마다 힙을 완전 해시로 체크포인트해서 시간여행/재실행을 만든다. WASM은 mprotect/dirty-page가 없어 실행 경계 해시로 델타를 재구성한다.
- **프로세스 OS(process-os)**: Web Worker = 프로세스, 스냅샷 = 프로세스 이미지, 워커 주입 = fork. 독립 인터프리터 N개 = 독립 GIL N개 = N코어 물리 병렬. WASM dlopen 문제를 회피하므로 오늘 가능한 최상단이다.

지원: **Chromium/Edge 전용** (JSPI + SharedArrayBuffer + crossOriginIsolated). Firefox/Safari 미지원은 결함이 아니라 스코프다.

## 언어

- 대화와 문서는 한국어. 코드 식별자, 라이브러리명, API 이름처럼 번역하면 흐려지는 기술 명칭은 원어 유지.

## 협업 스타일

- 다중 선택 survey 형태로 묻지 않는다. 분석 + 추천 + 근거 먼저, redirect 받기.
- 진짜 차단되는 결정 1개만 짧게 묻는 건 OK. 3-4개 동시 질문 금지.
- 아이디어 요청에는 enumerated 리스트로 충분히 나열. 골라달라는 질문 없이.

## 코드 규칙

- **네이티브 ESM, 빌드 단계 없음.** 번들러/트랜스파일러 도입 금지. `.js` 그대로 `import`.
- `camelCase` 파일/폴더/함수/변수, `PascalCase` 클래스.
- 소비자는 능력 계약(`MemoryCapability` 등)만 쓴다. `HEAPU8` 같은 엔진 내부를 컴포넌트에서 직접 만지지 않는다. 교차 관심사는 능력 뒤에 격리한다.
- `try/catch`로 삼키고 무시(`catch (e) {}`) 하는 패턴은 스택 복원 같은 best-effort 자리에만 국한한다. 나머지는 좁힌 예외 + 컨텍스트를 담아 재던진다.
- 백업성 루트(`_backup/`, `_archive/`, `_reference/`) 생성 금지. 보존할 가치가 있으면 정식 위치로 승격한다.
- 저장소 루트에는 제품/운영 파일만 둔다. 임시 산출물/로그/실습 파일은 커밋하지 않는다.

## 테스트

- `npm test` = `node tests/run.mjs`. Node 전용 구조/린트 게이트(공개 표면 존재·타입, em dash 0, AI 흔적 0)를 돌린다. 의존성 0.
- 브라우저 실측은 `examples/`의 HTML을 crossOriginIsolated 서버(COOP/COEP 헤더)로 띄워 확인한다. WASM 런타임 특성상 진짜 검증은 브라우저에서만 가능하다.
- 커밋 전 `npm test`가 green이어야 한다.

## Git 규칙

- **main 전용.** 로컬 브랜치 생성/푸시 금지. `git switch -c`, `git checkout -b`, `git branch <name>` 사용 안 함. `.githooks`의 `reference-transaction`과 `pre-push`가 non-main ref를 차단한다.
- 원격 반영은 `main -> origin/main`만.
- 훅 경로는 `.githooks`다. 클론 후 `git config core.hooksPath .githooks`로 활성화한다.

## 절대 게이트 (위반 시 즉시 차단)

- 커밋 메시지/주석/문서에 도구·생성 흔적 금지: `AI`, `GPT`, `OpenAI`, `ChatGPT`, `Codex`, `Claude`, `Anthropic`, `Generated by`, `Co-Authored-By`, `Assisted-by`. `.githooks/commit-msg`가 차단한다.
- 문서(`*.md`)와 소스에 em dash 문자(U+2014) 금지. 구분이 필요하면 하이픈(-), 쉼표, 문장 재구성. `.githooks/pre-commit`이 스테이징된 `*.md`/`*.js`에서 발견 시 커밋 차단.
- 버전은 `0.0.x` 라인. 릴리즈 때 끝자리만 1씩 올린다. `package.json`과 태그(`v0.0.x`)가 같은 값이어야 한다.

## 소비 계약 (products가 pyproc을 쓰는 법)

- 소비 제품은 **버전 태그로 고정**해서 가져간다(플로팅 금지). 브레이킹은 patch 노트에 명시.
- 제품 UI/도메인 로직은 pyproc에 넣지 않는다. pyproc은 런타임/능력만 제공하고, 제품은 그 위에 자기 표면을 얹는다.
- SSOT는 "참조"가 아니라 "실제 import"로만 성립한다. codaro가 first consumer이며, dartlab/xlpod는 점진 이관한다.
