# speculative-fleet - 투기적 탐색 프리미티브

> ✅ 완료 (2026-07-17): 실측(4.05배 방송 팬아웃, 5.2배 병렬 탐색)을 근거로 `forkMany`를
> 공개 프리미티브로 승격하고 게이트로 잠갔다. fork는 1:1 위임으로 계약 불변.

상태: 완료.

에이전트의 실제 루프는 "한 번 실패하고 한 번 되돌리기"가 아니라 **여러 후보를 동시에
시험하고 이긴 것만 남기기**다. 핵(결정적 리플레이 경계 + 페이지 델타)은 그 재료를 이미
갖고 있는데, 표면이 그걸 한 번에 하나(`fork(src, dst)`)로만 준다. 이 이니셔티브는
그 간극을 닫는다.

## 한 문장

**준비된 상태 하나를 한 번만 수확해 N 레인에 방송하고, 이긴 후보를 본선으로 승계한다.**

## 왜 지금

[branchFleet 캠페인](../../../tests/attempts/branchFleet/README.md)이 브라우저 실측 7/7로
가설을 세웠다(2026-07-17, 21.4MB 델타 / 본선 1 + 후보 4):

- 방송 팬아웃(수확 1회 + SAB) 78ms vs 순차 fork 4회 316ms = **4.05배**.
- 그 위 4-후보 병렬 탐색 90ms vs 직렬 재시도 468ms = **5.2배**, 결과는 직렬과 동일.
- 승계(역방향 fork 1회) 60ms 뒤 본선이 승자 상태와 정확히 일치, 함대 재사용 성립.

실측이 먼저 있고 계약이 뒤따르는 순서를 지켰다. 남은 일은 이 이득을 공개 표면의
프리미티브로 고정하고 게이트로 잠그는 것이다.
