# 00 - 계획

## 무엇을 만드나

**경계 챌린지(boundary challenge).** 매니페스트 하나를 주면 누구든 부팅해서 힙 페이지 해시
벡터를 내놓고, 서로 대조해서 **몇 페이지가 다른지**를 말하는 장치. `guix challenge`가 파일에
하는 것을 **부팅 후 힙**에 한다.

산출물은 서사가 아니라 **숫자 하나**다: `상이 페이지 = N`. N이 0이면 그 반경 안에서 경계가
같다는 최초의 증거고, N이 0이 아니면 **그 N개가 어디서 왔는지**가 산출물이다. 둘 다 성공이다.

## 왜 이게 1단계인가

독립 조사 넷이 각자 다른 문헌을 뒤지고 같은 문장에 도달했다.

> 서로 조율한 적 없는 참가자들이 각자 결정적으로 부팅해 **바이트 동일한 힙 경계**에 도달하고,
> 그 경계를 **한 번도 전송하지 않은 채** 그것을 증명하는 것. **NOT FOUND.**

> 정말로 안 돌려본 것: 엔진 빌드·아키텍처·메모리 압력의 **실제 타깃 매트릭스에 걸쳐 Pyodide
> 이미지를 해시하고 바이트를 비교하는 것.** **아키텍처가 아니라 그 실험이 기여로 보인다.**

> 매니페스트 -> 바이트 동일 이미지를 **제3자가 독립 검증**하는 보장은 출시된 어디에도 없다.
> 그게 가능한 가장 강한 주장이고, 그건 **스냅샷이 아니라 검증 가능성**에 관한 것이다.

> 진짜 틈은 서명이 아니라 **머신 상태의 결정론적 content address** 쪽이다.

넷 다 **측정**을 가리킨다. 그리고 넷 다 **아키텍처는 이미 남의 것**이라고 말한다. 우리가
새로 지을 것은 없고, 재볼 것이 있다.

## 두 번째 기기를 살 필요가 없다

이 계획의 유일한 비자명한 수다. **CI 러너가 두 번째 기기다.**

`npm run test:browser`가 이미 headless Chromium을 GitHub Actions에서 돌린다. 그건 다른 CPU고,
다른 커널이고, 다른 메모리 압력이고, 종종 다른 Chromium 빌드다. 해시 매니페스트를 저장소에
커밋하고 CI가 재계산해서 대조하면 **그게 기기 대 기기 측정이다.** 비용 0.

매니페스트 크기: 힙 40MB / 64KB 페이지 = 약 640 페이지 x 32바이트 = **20KB**. 커밋해도 된다.

## 단계

### 1단계: 측정 장치 (게이트 아님)

`tests/attempts/bootDeterminism/`(신설 캠페인) 안에서:

- **`challengeManifest`**: 무엇을 부팅했는지의 정본. Pyodide 버전, index URL, 자산 SRI,
  패키지 목록, 엔트로피 고정 항목. **이것이 "같은 매니페스트"의 정의다.** 지금은 그 정의가
  코드 어디에도 없다.
- **`heapHashVector`**: 부팅 -> 경계 -> 페이지별 sha256. 힙 길이와 페이지 크기도 싣는다.
  길이가 다르면 페이지 비교는 무의미하므로 먼저 길이를 본다.
- **`challengeCompare`**: 벡터 둘 -> `{ samePages, diffPages, diffIndices }`.

**측정하는 축(싼 것부터):**

| 축 | 방법 | 이미 아는 것 |
|---|---|---|
| 워커 대 워커 (같은 탭) | `forkLiveProbe` | **바이트 동일**(8/8) |
| 메인 대 워커 (같은 탭) | 기존 실측 재확인 | **다름**(`worker.js:10`) |
| 부팅 대 부팅 (같은 탭) | `bootDeterminismProbe` | 180p -> 엔트로피 고정 후 0p |
| **탭 대 탭 (같은 기기)** | 벡터를 파일로 내보내 두 번째 탭에서 대조 | **미측정** |
| **기기 대 기기** | **벡터를 커밋하고 CI가 재계산** | **미측정** |
| **Chromium 빌드 대 빌드** | CI가 두 버전 설치 | **미측정** |

**게이트로 만들지 않는다.** 결과를 모르는 것을 게이트로 만드는 것은 규칙 위반이다(신설
게이트는 음성 시험으로 이빨을 증명하고서야 게이트다). 1단계는 숫자를 얻는 것으로 끝난다.

### 2단계: 발산 출처 특정

N이 0이 아니면 `diffIndices`를 들고 **그 페이지에 무엇이 사는지** 찾는다. 힙 주소 ->
파이썬 객체 역추적은 이미 `heapDelta` 실측에서 해봤다. 후보는 미리 있다.

- **포인터/주소가 힙에 그대로 직렬화됨.** Node/V8가 재현성 작업에서 최초로 만난 벽이 이것이었다
  ("메모리 주소가 실행마다 바뀌었다"). WASM 선형 메모리는 ASLR이 없지만 **할당 순서**는
  결정적이어야 한다.
- **초기화되지 않은 struct padding.** V8이 placement new 전에 0으로 밀어서 고쳤다.
- **타임스탬프.** `SOURCE_DATE_EPOCH`의 힙판.
- **hiwire 슬롯.** Cloudflare가 명시한 것: **선형 메모리가 상태의 전부가 아니다.** 함수 포인터
  테이블과 JS 객체 테이블이 밖에 산다. 우리 벽(#5195, `index 7`)과 같은 자리다.

이 목록 자체가 산출물이다. **아무도 파이썬 힙에 대해 이 목록을 안 만들었다.**

### 3단계: 반경을 계약으로

측정된 것만 문서에 박는다. 능력 매트릭스와 API 문서가 "경계는 이 반경 안에서만 같다"를 말한다.
지금은 아무 데도 그 한정이 없고, 그래서 `fork`가 워커끼리만 되는 이유를 소비자가 알 방법이 없다.

### 4단계: 게이트

**이 단계는 1단계 결과에 종속이다.** 0이 나온 축만 게이트가 된다. 0이 아닌 축은 게이트가
아니라 **계약 실태 표의 항목**이 된다. 순서를 뒤집지 않는다.

### 5단계: 우리가 소유한 경계(WASI)도 같은 자로

WASI 레인은 페이지 결정성이 **미측정**이다(파이썬 가시 상태로만 쟀다). 체크포인트가 힙 40MB
통째 복사다(`wasiWorker.js:77`). **논지가 우리 엔진에 없다.** 같은 벡터 장치를 물린다.

### 6단계: `PYTHONHASHSEED=0` 하자

`bootSession`이 이걸 하드코딩하고 CPython은 인터프리터 초기화 때 읽으므로 **세션 내내 hash
randomization이 꺼진다**(CVE-2012-1150 대응책 무력화).

**이게 순진한 절반이라는 것이 조사로 확정됐다.** V8은 같은 문제를 **고정 + 역직렬화 때 rehash**로
푼다: 빌드 때 `--random_seed`로 재현성을 얻고, 복원 때 새 시드를 뽑아 관련 필드를 전부 rehash해서
hash flooding 내성을 되찾는다. **결정성과 신선한 엔트로피는 배타적이지 않다.** ART도 2016년에
"identity hashcode seed를 고정해야 한다"를 같은 이유로 했다.

CPython엔 rehash 설비가 없다. 그러니 지금 할 수 있는 것은 **트레이드오프를 기록하는 것**이고,
그것이 이 단계의 전부다. 위협 모델상 피해가 자기 탭에 국한된다는 것도 함께 적는다.

## 완료 조건

1. **반경이 수치로 있다.** 탭 대 탭과 기기 대 기기(= CI) 두 축의 상이 페이지 수가 나온다.
2. **0이 아니면 출처가 특정된다.** `diffIndices`가 이름을 얻는다.
3. **반경이 문서에 박힌다.** 능력 매트릭스와 API 문서가 한정을 계약으로 말한다.
4. **측정된 것만 게이트가 된다.** 0인 축만.
5. **WASI 레인도 같은 자로 잰다.**
6. **`PYTHONHASHSEED=0` 트레이드오프가 계약 실태 표에 오른다.**

## 완료 조건이 아닌 것

- **"경계는 공공재다"를 주장하지 않는다.** JVM CDS가 2019년부터 하고 있고 Cloudflare workerd가
  `baseline` + `dedicated` stacked snapshot으로 **출시했다**. 우리 반경은 아직 워커 하나다.
- **`forkMany`가 새롭다고 하지 않는다.** SnowFlock이 2009년에 "한 번 수확해서 N에 방송"을
  측정하고 **열등하다고 기각**했다(157.29s vs 자기 설계 70.63s). 우리는 그들이 버린 baseline에
  서 있다. 이건 결함이 아니다(브라우저엔 IP 멀티캐스트도 CoW도 없다). 다만 **자랑이 아니다.**
- **속도 우위를 주장하지 않는다.** 공개 표면 숫자 자랑은 강행규칙이 금지한다.
- **경계를 배포하지 않는다.** 반경을 재기 전에는 배포할 대상이 정의되지 않는다.
- **기본 엔진을 바꾸지 않는다.** `python-3.14.6.wasm`은 export가 2개(`memory`, `_start`)라
  동기 `runSync` 호출 지점이 없고, 메모리가 `shared=false`라 메인스레드에서 `heapU8()`을 못 잡는다.
